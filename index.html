PT-P2 kNN Predictive Modelling

Lab Exercise PT-P2 (Initial Predicition Model - kNN)

Original Code and Execution

import matplotlib.pyplot as plt
from sklearn.datasets import load_iris

# Load data
data = load_iris()

# Separate features and target
X = data.data
y = data.target

# Splitting the data for training and testing
from sklearn.model_selection import train_test_split
X_Train, X_Test, y_Train, y_Test = train_test_split(X, y, test_size = 0.25, random_state = 5)

# Create a KNeighborsClassifier object
from sklearn.neighbors import KNeighborsClassifier
knn = KNeighborsClassifier(n_neighbors = 5)

# Train the model using the training data
knn.fit(X_Train, y_Train)

# Use the model to predict the test set
predictions = knn.predict(X_Test)

# Check accuracy
from sklearn.metrics import accuracy_score
accuracy = accuracy_score(y_Test, predictions)
print("Accuacy :", accuracy)

# Evaluate model performance
from sklearn.metrics import classification_report
print("classification report")
print(classification_report(y_Test, predictions))

# Making a confusion matrix
from sklearn.metrics import confusion_matrix
cm = confusion_matrix(y_Test, predictions)

# Visualization of the Scatter plot with predicted classes
plt.figure(figsize=(8, 6))
for i in range(3):
  plt.scatter(X_Test[:,0][y_Test == i], X_Test[:, 1][y_Test == i], cmap='viridis')
plt.scatter(X_Test[:, 0], X_Test[:, 1], c=predictions, cmap='viridis', alpha=0.8)
plt.xlabel(data.feature_names[0])
plt.ylabel(data.feature_names[1])
plt.title("Scatter Plot with Predcted Classes")
plt.legend()
plt.show()

# Confusion matrix visualization
plt.figure(figsize=(8, 6))
plt.imshow(cm, cmap='Blues')
plt.colorbar()
plt.title("Confusion Matrix")
plt.xlabel("Prediction Label")
plt.ylabel("True Label")
plt.xticks(ticks=range(3), labels=data.target_names)
plt.yticks(ticks=range(3), labels=data.target_names)
plt.show()

print("Confusion matrix :")
print(cm)

Code Organization and Explanation

Setup & Imports

import matplotlib.pyplot as plt
from sklearn.datasets import load_iris

The code presented above is used to access external libraries that are available for your program. It is usually a good practice to place these at the very top of the code to help other programmers understand which imports or libraries are being utilized. This also helps keep the code clean and avoids confusion.

The given code utilizes import matplotlib.pyplot with the alias plt, which enables plotting and visualization. On the other hand, from sklearn.datasets import load_iris loads a sample machine learning dataset.

There are many available libraries you can use, such as Plotly, Seaborn, and others. Make sure to use only the libraries your program actually needs, because fewer imports result in faster program startup times, and unused libraries can take up memory.

Data Acquisition

# Load data
data = load_iris()

# Separate features and target
X = data.data
y = data.target

The code above is used as a dictionary containing a set of input data (features) and target data (labels). Machine learning models cannot work without this data. The load_iris() function loads the dataset into memory and stores it in a variable called data.

We separate X and y because X is fed into the model as input, and the model’s predicted results are compared to y. Without X, the model cannot learn. Without y, the model cannot check whether its predictions are correct.

Train/Test Splitting and Model Building & Training

# Splitting the data for training and testing
from sklearn.model_selection import train_test_split
X_Train, X_Test, y_Train, y_Test = train_test_split(X, y, test_size = 0.2, random_state = 0)

# Train the model using the training data
knn.fit(X_Train, y_Train)

We split the data to evaluate the model’s performance on unseen data. This is done to train the model more accurately and fairly. If we train the system using the same data it was evaluated on, overfitting can occur, where the model does not truly learn but instead memorizes the data.

The train_test_split line randomly divides the dataset and stores the results into four (4) variables: X_train, X_test, y_train, and y_test. The test size is set to 20% to maintain a balance between having enough data for training and testing. Other common values include 0.25 and 0.3. Note: the higher the test_size the lower the accuracy result. The random_state parameter acts as a controller for the randomness of the split. Without it, each run may produce different results, which can affect accuracy.

Moreover, setting the number of neighbors (k) to 5 means the model looks for the five (5) nearest training points and decides the class based on the majority vote. Afterward, the .fit() function stores the training data. At this stage, no explicit equations are learned. Instead, the model now knows the training data and can compute distances, find nearest neighbors, and vote on class labels when making predictions.

Prediction and Evaluation Metrics

In these lines, the code reports the overall accuracy of the model, including the classification report and error distribution. The .predict() function generates the predicted labels, while accuracy_score compares the predicted values with the actual labels to compute the overall correctness of the model.

The classification report summarizes performance metrics for each class, such as precision, recall, and F1-score. The confusion matrix is used to show the count of true versus predicted class outcomes, helping to visualize where the model makes correct predictions and where it makes errors.

In these lines, the code creates visualizations such as a scatter plot and a heatmap to illustrate the results. The scatter plot gives a quick sense of class separation, while the heatmap shows which classes are confused with each other.

The plt.scatter(x, y, c=labels, cmap, alpha) parameters are used to color the points by class. We plot the test set using two features and include a legend to make the class labels readable.

On the other hand, plt.imshow(cm, cmap='Blues') displays the confusion matrix as an image, while plt.xticks() and plt.yticks() assign class names to the axes for better clarity.

Output Analysis and Experiment

The purpose of this program is to build, train, and evaluate a K Nearest Neighbors (KNN) classification using the Iris dataset. It is trying to predict species of an Iris flower which is the Iris-setosa, Iris-versicolor, and Iris-virginica represented by y and four numerical features (Sepal length, Sepal width, Petal Length, and Petal Width).

The K Nearest Neighbor determines how many nearby data the model compare — it can be classified into 3 classes: Small k (1–3) this model is highly sensitive to individual data, Moderate k (5–7) is the balance, and Large k (10–15) becomes more efficient but may underfit which ignores the important patterns. Moreover, the misclassification occurs between the Iris-versicolor and Iris-virginica because these two species have overlapping features.

WW-P2 Data Cleaning

Code For Data Cleaning Pipeline

import pandas as pd
import numpy as np
from sklearn.preprocessing import MinMaxScaler

df = pd.read_csv('adjusted_noisy_knn_300.csv')

df = df.drop_duplicates(subset=['Customer_ID'])

df.loc[(df['Age'] > 100) | (df['Age'] < 0), 'Age'] = np.nan
df.loc[df['Annual_Income'] < 0, 'Annual_Income'] = np.nan

df['Age'] = df['Age'].fillna(df['Age'].median())
df['Annual_Income'] = df['Annual_Income'].fillna(df['Annual_Income'].median())
df['Total_Spend'] = df['Total_Spend'].fillna(df['Total_Spend'].median())

df['Gender'] = df['Gender'].str.strip().str.lower().replace({
    'm': 'Male', 'male': 'Male', 'f': 'Female', 'female': 'Female', 'fem': 'Female'
})
df['Device_Type'] = df['Device_Type'].str.strip().str.capitalize()

df['Purchase_Volume'] = pd.cut(df['Items_Purchased'],
                              bins=[0, 15, 35, 51],
                              labels=['Low', 'Medium', 'High'])

region_tax_data = pd.DataFrame({
    'Store_Region': ['North', 'South', 'East', 'West'],
    'Tax_Rate': [0.05, 0.07, 0.06, 0.08]
    })
df = pd.merge(df, region_tax_data, on='Store_Region', how='left')

scaler = MinMaxScaler()
numeric_cols = ['Age', 'Annual_Income', 'Total_Spend', 'Items_Purchased',
                    'Last_Logon_Days', 'Support_Calls', 'Tax_Rate']
df[numeric_cols] = scaler.fit_transform(df[numeric_cols])

df = pd.get_dummies(df, columns=['Gender', 'Device_Type', 'Store_Region', 'Purchase_Volume'])

cols_to_drop = ['Membership_Score', 'Newsletter_Sub', 'Customer_ID', 'Purchase_Date']
df_final = df.drop(columns=cols_to_drop)

# Save the prepared dataset
df_final.to_csv('cleaned_knn_data.csv', index=False)
print("Data Preparation Complete. Final shape:", df_final.shape)

import pandas as pd
import numpy as np

# LOAD DIRTY DATA
df = pd.read_csv("/content/dirty_cafe_sales.csv")
df = df.drop(columns=["Transaction ID"], errors="ignore")

# PLACEHOLDERS
PLACEHOLDERS = ["ERROR", "UNKNOWN", "N/A", "NA", "null", "None", "", "nan", "NaN"]

# CLEAN NUMERIC
num_cols = ["Quantity", "Price Per Unit", "Total Spent"]
for c in num_cols:
    df[c] = df[c].astype(str).str.strip().replace(PLACEHOLDERS, np.nan)
    df[c] = pd.to_numeric(df[c], errors="coerce")

# SMART IMPUTATION: Total = Quantity * Price
mask_total = df["Total Spent"].isna() & df["Quantity"].notna() & df["Price Per Unit"].notna()
df.loc[mask_total, "Total Spent"] = df.loc[mask_total, "Quantity"] * df.loc[mask_total, "Price Per Unit"]

mask_qty = df["Quantity"].isna() & df["Total Spent"].notna() & df["Price Per Unit"].notna() & (df["Price Per Unit"] != 0)
df.loc[mask_qty, "Quantity"] = df.loc[mask_qty, "Total Spent"] / df.loc[mask_qty, "Price Per Unit"]

mask_price = df["Price Per Unit"].isna() & df["Total Spent"].notna() & df["Quantity"].notna() & (df["Quantity"] != 0)
df.loc[mask_price, "Price Per Unit"] = df.loc[mask_price, "Total Spent"] / df.loc[mask_price, "Quantity"]

# Quantity should be integer-like
df["Quantity"] = np.round(df["Quantity"]).astype("Int64")

# Median fallback
for c in num_cols:
    df[c] = df[c].fillna(df[c].median())

# CLEAN CATEGORICAL
cat_cols = ["Item", "Payment Method", "Location"]
for c in cat_cols:
    if c in df.columns:
        df[c] = df[c].astype(str).str.strip().replace(PLACEHOLDERS, "Unknown")

# DATE → FEATURES
df["Transaction Date"] = pd.to_datetime(df["Transaction Date"], errors="coerce")
if df["Transaction Date"].notna().any():
    df["Transaction Date"] = df["Transaction Date"].fillna(df["Transaction Date"].mode()[0])
df["trans_month"] = df["Transaction Date"].dt.month
df["trans_day"]   = df["Transaction Date"].dt.day
df["trans_dow"]   = df["Transaction Date"].dt.dayofweek
df = df.drop(columns=["Transaction Date"])

# OPTIONAL: drop Payment Method
if "Payment Method" in df.columns:
    df = df.drop(columns=["Payment Method"])

# REMOVE DUPLICATES
before = len(df)
df = df.drop_duplicates()
after = len(df)
print(f"Duplicates removed: {before - after}")

# OUTLIER HANDLING (Winsorize)
Q1 = df["Total Spent"].quantile(0.25)
Q3 = df["Total Spent"].quantile(0.75)
IQR = Q3 - Q1
lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR
outliers_count = ((df["Total Spent"] < lower) | (df["Total Spent"] > upper)).sum()
print(f"Outliers detected (Total Spent): {outliers_count}")
df["Total Spent"] = df["Total Spent"].clip(lower, upper)

# SAVE CLEANED
df.to_csv("/content/cleaned_cafe_sales.csv", index=False)
